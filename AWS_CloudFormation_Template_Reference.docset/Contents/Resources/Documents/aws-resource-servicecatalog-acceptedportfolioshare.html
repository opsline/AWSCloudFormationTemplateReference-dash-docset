AWS::ServiceCatalog::AcceptedPortfolioShare

Accepts an offer to share the specified portfolio for AWS Service Catalog. For more information, see AcceptPortfolioShare in the AWS Service Catalog Developer Guide.

Topics

  • Syntax
  • Properties
  • Return Values

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "Type" : "AWS::ServiceCatalog::AcceptedPortfolioShare",
  "Properties" : {
    "AcceptLanguage" : String,
    "PortfolioId" : String
  }
}

YAML

Type: "AWS::ServiceCatalog::AcceptedPortfolioShare"
Properties:
  AcceptLanguage: String
  PortfolioId: String

Properties

AcceptLanguage

The language code.

Required: No

Type: String

Update requires: Replacement

PortfolioId

The portfolio identifier.

Required: Yes

Type: String

Update requires: Replacement

Return Values

Ref

When you pass the logical ID of an AWS::ServiceCatalog::AcceptedPortfolioShare resource to the intrinsic Ref function, the function returns a unique identifier.

For more information about using the Ref function, see Ref.
